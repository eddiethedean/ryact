## schedulyr user guides

These guides are focused on using `schedulyr` as a library (not on porting work or upstream parity mechanics).

- `core_scheduler.md`: The cooperative `Scheduler` API (`schedule_callback`, `cancel_callback`, `run_until_idle`) and recipes.
- `production_scheduler.md`: The production-style `unstable_*` APIs (`production_scheduler.py`) and profiling surface.
- `host_harnesses.md`: Browser/mock/fork harnesses used to mirror upstream host behavior.
- `cross_runtime_suite.md`: Optional cross-runtime recording and comparisons against upstream `facebook/react`.
